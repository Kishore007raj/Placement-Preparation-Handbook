# Theory

## Definition

A string is a sequence of characters used to represent text.

## Why it exists

Strings allow programs to process human-readable data such as names, sentences, and file contents.

## Intuition

Every character has a position, and operations often depend on comparing or transforming these positions.

## Real-world analogy

A string is like a sentence written on a line of paper, where each word or character has a position.

## Working principle

- Characters are stored in order.
- Indexing helps access individual characters.
- Many operations depend on comparing adjacent characters or scanning the string.

## Advantages

- Easy to represent and read
- Widely used in input/output and parsing
- Flexible with built-in operations

## Disadvantages

- Some operations can be costly if done repeatedly
- Mutable behavior depends on language implementation

## Applications

- Text validation
- Search and replace
- Compression and encryption
- Parsing logs and data files

## Edge cases

- Empty string
- Single-character string
- Whitespace-only input
- Case sensitivity

## Interview notes

String problems often hide array-like patterns such as sliding window, hashing, and two pointers.
