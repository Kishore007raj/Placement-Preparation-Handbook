# 📑 Templates

Templates are reusable implementations of commonly used algorithms and data structures. They help you solve problems faster by eliminating repetitive coding during coding interviews and competitive programming contests.

This section contains clean, optimized, and well-tested C++ templates that can be reused across coding platforms and real-world problem-solving scenarios.

- Build your own competitive programming template.
- Reduce coding time during contests and interviews.
- Reuse battle-tested implementations.
- Avoid common implementation mistakes.
- Focus on problem-solving instead of boilerplate code.

# 📚 Template Categories

| Category | Description |
|----------|-------------|
| Global CP Template | Standard competitive programming template with macros, aliases, debugging utilities, and helper functions. |
| Fast I/O | Fast input/output techniques for handling large datasets efficiently. |
| Math Templates | GCD, LCM, modular arithmetic, fast exponentiation, combinatorics, and number theory utilities. |
| Graph Templates | BFS, DFS, Dijkstra, Bellman-Ford, Floyd-Warshall, MST, Topological Sort, and Union-Find. |
| String Templates | KMP, Z Algorithm, Rabin-Karp, Rolling Hash, Trie, and string-processing utilities. |
| Range Query Templates | Prefix Sum, Fenwick Tree, Segment Tree, Sparse Table, and range query/update techniques. |
| Arrays | Prefix sums, difference arrays, coordinate compression, and common array utilities. |
| Binary Search | Standard binary search, lower/upper bound, and binary search on answer. |
| DP | Common dynamic programming templates and optimization techniques. |
| DSU | Disjoint Set Union (Union-Find) with path compression and union by rank/size. |
| Fenwick | Binary Indexed Tree templates for efficient updates and queries. |
| Segment Tree | Segment Tree implementations with lazy propagation and variations. |
| Sliding Window | Fixed-size and variable-size sliding window templates. |
| Trie | Trie implementations for strings and bitwise operations. |
| Two Pointers | Same-direction and opposite-direction pointer templates. |
| Examples | Sample problems demonstrating how to use the templates. |

# 🚀 Why Use Templates?

Well-designed templates save valuable time by eliminating repetitive implementation work. Instead of rewriting the same code, you can focus on understanding the problem and choosing the right approach.

Templates help you:

- Write code faster.
- Reduce implementation bugs.
- Maintain consistent coding style.
- Reuse proven solutions across problems.
- Perform better in contests and interviews.
