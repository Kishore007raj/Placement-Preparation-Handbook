# Theory

## Definition

An array is a linear data structure that stores elements of the same type in contiguous memory locations.

## Why it exists

Arrays provide fast access to elements using indices, making them ideal for sequential data storage.

## Intuition

If you know the position of an item, you can access it directly without scanning the whole structure.

## Real-world analogy

A row of lockers is like an array: each locker has a fixed position.

## Working principle

- Elements are stored at consecutive addresses.
- The index of an element determines its position.
- Access time is constant for direct lookup.

## Advantages

- Fast indexing
- Memory-efficient for fixed-size data
- Easy to traverse

## Disadvantages

- Fixed size in many languages
- Insertion and deletion can be costly

## Applications

- Storing sensor values
- Implementing queues and stacks
- Building lookup tables

## Edge cases

- Empty array
- Single element
- Duplicate values
- Out-of-bounds access

## Interview notes

Arrays are the foundation for many advanced patterns such as sliding window, prefix sums, and two pointers.
