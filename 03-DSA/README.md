# 📚 Data Structures

Data structures define how data is organized, stored, and accessed. Choosing the right data structure is often the first step toward writing an efficient algorithm.

This section covers the core data structures used in software engineering interviews, competitive programming, and everyday problem solving. Each topic focuses on understanding how the data structure works, how to implement it from scratch, and when to use it.

---

# 🎯 Learning Objectives

By completing this section, you will be able to:

- Choose the right data structure for a given problem.
- Understand how each data structure works internally.
- Implement every data structure from scratch.
- Analyze time and space complexity.
- Solve common interview problems confidently.

# 📊 Complexity Cheat Sheet

| Data Structure | Access | Search | Insert | Delete |
|----------------|:------:|:------:|:------:|:------:|
| Array | O(1) | O(n) | O(n) | O(n) |
| String | O(1) | O(n) | O(n) | O(n) |
| Hash Table* | — | O(1) | O(1) | O(1) |
| Linked List** | O(n) | O(n) | O(1) | O(1) |
| Stack | O(n) | O(n) | O(1) | O(1) |
| Queue | O(n) | O(n) | O(1) | O(1) |
| Heap | — | O(n) | O(log n) | O(log n) |
| Binary Search Tree*** | O(log n) | O(log n) | O(log n) | O(log n) |
| Graph | Depends on representation | O(V + E) | Varies | Varies |
| Trie | O(L) | O(L) | O(L) | O(L) |

> **Notes**
>
> - *Average-case complexity for hash tables.
> - **Insertion and deletion assume a reference to the node is already available.
> - ***For balanced Binary Search Trees.
