# 01 - Arrays & Hashing

Complete prep folder for the **Arrays & Hashing** topic - built for freshers preparing for company OAs and interview DSA rounds. Language: **C++**.

## Folder Contents

| File | Purpose | Read When |
|---|---|---|
| `Theory.md` | Fundamentals of arrays & hashing (how they work, STL internals, complexity basics) | First - build the foundation |
| `Patterns.md` | The 10 core patterns, each with Name / When to Use / Tips / Recognition / Template / Complexity / Constraints | Main study material |
| `Templates.cpp` | All C++ code templates in one copy-paste-ready file | While practicing / during OA |
| `Cheatsheet.md` | One-page quick reference + decision flowchart | Quick lookup, right before a round |
| `Complexity.md` | Time & space complexity table for every pattern + general rules of thumb | When comparing approaches |
| `Interview_Notes.md` | High-yield problem list, company OA tips, common fresher mistakes | Before interviews specifically |
| `Revision.md` | Night-before-interview recall sheet + personal mistake log | Last-minute revision |

## The 10 Patterns Covered
1. Prefix Sum
2. Two Pointers
3. Sliding Window (Fixed Size)
4. Sliding Window (Variable Size)
5. Fast & Slow Pointers / Cyclic Sort
6. Hashing - Frequency Counting
7. HashMap - Complement / Pair Search
8. Kadane's Algorithm (Maximum Subarray)
9. Dutch National Flag (3-Way Partition)
10. Monotonic Stack

## Suggested Study Order
1. Read `Theory.md` once - understand *why* arrays and hash maps behave the way they do.
2. Go through `Patterns.md` pattern by pattern. For each: read Recognition → understand Template → note Constraints.
3. Type out each template from `Templates.cpp` from scratch (don't copy-paste) until it's memorized.
4. Solve the "High-Yield List" problems in `Interview_Notes.md`, two passes each (brute force, then optimal).
5. Use `Cheatsheet.md` as your revision doc during timed practice.
6. Night before an interview/OA: read only `Revision.md`.

## How to Use This Folder Effectively
- Don't just read - **type every template by hand** at least 3 times each. Muscle memory matters more than reading speed under time pressure.
- After solving any array/hashing problem (from LeetCode/GFG/company OA), come back and add one row to the mistake log in `Revision.md`.
- If you get stuck deciding a pattern in a live problem, mentally run through the "Red Flags" trigger list in `Revision.md` - this is designed to be recalled in under 30 seconds.
